# Project Instructions: The Rules Your Project Runs On

If a regular chat is a sticky note, project instructions are the label on the whole filing cabinet.

That is the big idea. Instead of repeating yourself in every new message, you give ChatGPT a standing set of instructions for that project. It keeps the work more consistent, less chaotic, and far less likely to wander off like a GPS from 1998.

## Why this matters

Beginners often run into the same problem: they start with a good prompt, get a decent answer, then five messages later the tone changes, the format changes, and suddenly the AI is doing interpretive dance instead of following directions.

Project instructions solve that.

They let you set the default rules for an entire project, such as:

- your goal
- your preferred tone
- your audience
- formatting rules
- things to avoid
- how you want answers structured

So instead of saying, "Make this concise, professional, and in bullet points" every single time, you set that once at the project level.

## The idea in plain English

Project instructions are persistent guidance for one specific project.

Think of them as the house rules. Every chat inside that project works within those rules unless you clearly override them in a specific prompt.

Good project instructions usually include:

| Part | What to include |
|---|---|
| Goal | What this project is for |
| Audience | Who the content or work is aimed at |
| Tone | Formal, friendly, concise, persuasive, etc. |
| Format | Bullets, tables, summaries, sections, checklists |
| Constraints | Word limits, banned phrases, must-include items |
| Workflow | How you want ChatGPT to behave, such as asking clarifying questions first |

The key is this: project instructions should guide the work broadly, not try to cram every possible task into one giant mega-rulebook. If it starts looking like a fax from 1993, it is probably too much.

## Simple example

Let’s say you create a project for writing a weekly team update.

Your project instructions might say:

- Write in a clear, professional tone
- Keep updates under 200 words
- Use short sections: wins, risks, next steps
- Avoid jargon
- If information is missing, ask a clarifying question before drafting

Now every chat in that project starts from those assumptions. You can simply say, "Draft this week’s update from these notes," and ChatGPT already knows the style and structure you want.

## Quick tips

- Keep instructions short enough to scan quickly.
- Focus on repeatable rules, not one-time requests.
- Include both what you want and what to avoid.
- Update the instructions if the project changes direction.
- Use project instructions for consistency, then use individual prompts for specific tasks.

## Quick summary

Project instructions are the built-in rules for a ChatGPT project. They save time, reduce repetition, and keep responses more consistent across multiple chats. Set the goal, audience, tone, format, and constraints once, then let your prompts focus on the actual work.
